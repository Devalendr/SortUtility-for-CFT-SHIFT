Версия Java - SDK 11.0.16
Сборка Maven 4.0
Дополнительные библиотеки: org.apache.commons.cli версии 1.6 https://commons.apache.org/cli/download_cli.cgi

Main файл находится по пути SortUtility/src/main/java/Main

В папке содержится исходный код программы сортировщика, по пути out\artifacts\SortUtility_jar существует готовый файл для запуска через командную строку. 
По-умолчанию опцией -n или --name задаются имена файлов в папке SortUtility, при запуске через IDE, при запуске через командную строку/терминал файлы ищутся в папке, где находится .jar файл.
Опция -h или --help выводит на экран помощь по всем возможным опциям и примеры, если требуются.
Пример: java -jar SortUtility.jar -n in1.txt in2.txt
С помощью опции -o или --out можно задать путь к папке, куда будут помещены файлы с результатами, если опция отсутсвует, результаты помещаются в ту же папку, откуда были считаны файлы опции -n.
Пример: java -jar SortUtility.jar -o D:\sample_dir
Опцией -p или --prefix можно задать префикс для файла. 
Пример: java -jar SortUtility.jar -p result
Имена файлов будут: result_ints.txt, result_floats.txt, result_strings.txt.
Опцией -a или --add можно выбрать, будут ли перезаписываться файлы, или информация будет добавлена в них, если такая опция была встречена.
Опции -s --short и -f -- full соответсвенно можно выбрать отображение статистики в командной строке/терминале.


Пример запуска утилиты: java -jar SortUtility.jar -n in1.txt in2.txt -s -p res
Будут обработаны файлы in1.txt и in2.txt, в командную строку будут выведены данные короткой статистики, а файлы с результатами будут носить названия res_ints.txt, res_floats.txt, res_strings.txt

Пример запуска утилиты: java -jar SortUtility.jar -n in1.txt in2.txt -f -o D:\sample_dir -a
В результате обработаются файлы in1.txt и in2.txt, в командную строку будет выведена полная статистика, результаты будут записаны в файлы папки по пути D:\sample_dir с добавлением к уже существующим значениям.
